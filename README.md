## 🎮 The Premise

You're the main character. The world just got flooded with AI, everyone's panicking, and you've decided to do the smart thing: **become the person who builds the AI instead of fearing it.**

This repo is your campaign map. It's a **12-level journey** (0 → 11) from total beginner to *unstoppable AI engineer*. Every level follows the same loop:

> ### 📚 Learn → 🔗 Resources → ⚔️ Boss Project (ship it to GitHub)

By the end you won't just *know* things — you'll have a **GitHub profile full of real projects** that screams "hire me" without you saying a word.

No PhD required. No $20k bootcamp. Just you, a laptop, and stubbornness. 💪

## 🧙 Choose Your Character

Two paths up the same mountain. Pick the one that fits you **today** — you can always glance at the other.

| | 🧑‍💻 **Track A — The Coder** | 🌱 **Track B — The Newcomer** |
|---|---|---|
| **You are** | Already comfortable writing code (any language) | Never (or barely) written a line of code |
| **Your edge** | Speed — you skip the basics and sprint to AI | A clean slate — no bad habits to unlearn |
| **Start here** | 👉 **[tracks/with-coding.md](tracks/with-coding.md)** | 👉 **[tracks/no-coding.md](tracks/no-coding.md)** |

Both tracks **converge at Level 5** (Vibe Coding) and stay merged through the LLM, RAG, gen-media, agents, and deployment levels — once you're directing AI and talking to models, everyone's on equal footing. The boss projects are shared.

> 💡 **Not sure?** If you can write a `for` loop and a function without Googling, take Track A. Otherwise, Track B will treat you kindly.

## 🧭 How To Use This Repo

1. **Pick your track** (above).
2. **Work one level at a time.** Don't skip the boss project — *the project is the point.* Knowledge fades; a shipped repo is forever.
3. **Make every project a public GitHub repo** with a real README. This *is* your resume.
4. **Check the box** when you finish a level (below). Dopamine is a legitimate learning strategy. ✅
5. **Repeat until unstoppable.**

> ⏱️ **Pace:** There's no clock. Some people blitz this in 4 months full-time; others take 18 months on evenings and weekends. Both finish. The only failure is stopping.

## 🧱 The "GitHub Resume" Philosophy

Recruiters skim. Hiring managers want *proof*. A degree says "I sat in rooms." A GitHub full of working AI projects says **"I build things that work."**

So every level here ends with a project designed to be **shown, not just done**:

- ✅ Public repo, clear README with a screenshot or GIF
- ✅ A one-line "what this does" and a "how to run it"
- ✅ Live demo link when possible (Streamlit/HF Spaces/Vercel are free)
- ✅ Pin your best 6 on your profile

Full playbook in **[PROJECTS.md](PROJECTS.md)** → the "Turn Projects into a Resume" section.
